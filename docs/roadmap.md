# Roadmap

## Current Direction
Evolving the MVP into a stable, testable product (MVP v2) with refined voice interaction mechanics, decoupled word/sentence handling, and comprehensive architectural documentation (Static, Dynamic, and Deployment views).

## Milestone Roadmap

| Milestone | Timeline | Key Deliverables |
|---|---|---|
| **MVP v1** | Sprint 1 | Core voice recognition (Web Speech API), basic UI, 2 initial games (Coloring Game, Spell Caster), and GitLab Pages deployment without backend. |
| **MVP v2 (Initial)** | Sprint 2 | • **Product:** Refactored voice mechanics ("select-then-speak"), integrated dynamic word lists, new game prototypes.<br>• **Quality & Automation:** Established ISO/IEC 25010 Quality Requirements, Playwright E2E tests in CI, and Definition of Done. |
| **MVP v2.1 (Current)** | Sprint 3 | • **Product:** Decouple words and sentences logic (generic parsing handling) and add TTS audio pronunciation to word lists.<br>• **Architecture:** Full documentation of architecture views (Static, Dynamic, Deployment) and Configuration Management. |
| **MVP v3 (Next)** | Sprint 4 | • **Product:** Interactive background mechanics (Voice Angler), UI animation polish (Spell Match-3), and sentence power mechanic (Heroes of Voice & Magic).<br>• Finalize the remaining mini-game prototypes to reach the 10-game goal.<br>• **Infrastructure:** Migrate hosting to GitLab Pages. |
| **MVP v4 (Future)** | Future | • Parent progress dashboard (Local Storage), visual hints, and more complex game mechanics.<br>• Gamification features (stars/badges), advanced phrase processing. |
